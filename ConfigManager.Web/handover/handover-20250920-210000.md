## KEY DESIGN DECISION
- **Concept shift: category → group**: Use "group" concept instead of "category"
- Groups represent related configs that form a configuration object for apps
- Examples: `mysql` group (`myapp:mysql:host`, `myapp:mysql:port`), `redis` group
- This aligns better with how applications consume configuration sets
